# 신고 및 제재 알고리즘

## 1. 경고 → 자동 이용제한

**규칙**: 경고 **3회** 누적 시 자동으로 이용제한 **3일** 적용

**위치**: `UserSanctionService.addWarning()`

**상수**:
```java
WARNING_THRESHOLD = 3;      // 경고 3회 시 자동 이용제한
AUTO_SUSPENSION_DAYS = 3;  // 자동 이용제한 기간 (일)
```

**흐름**:
1. `UserSanction` (WARNING) 생성
2. `usersRepository.incrementWarningCount(userId)` — **원자적 증가** (동시성 해결)
3. `user.getWarningCount() >= 3` → `addSuspension(userId, "경고 N회 누적으로 인한 자동 이용제한", ..., 3)`

---

## 2. 이용제한 추가 (addSuspension)

- `UserSanction` (SUSPENSION) 생성
- `user.status = SUSPENDED`, `user.suspendedUntil = now + days`

---

## 3. 영구 차단 (addBan)

- `UserSanction` (BAN) 생성
- `user.status = BANNED`, `suspendedUntil = null`

---

## 4. 만료된 이용제한 자동 해제

**위치**: `UserSanctionService.releaseExpiredSuspensions()` (스케줄러 호출)

**흐름**:
1. `findExpiredSuspensions(now)` — `endsAt <= now` 인 SUSPENSION 조회
2. 각 유저별 `findActiveSanctionsByUserId` — 활성 제재 확인
3. **영구 차단** 있으면 → 해제 안 함
4. **활성 이용제한** 없으면 → `status = ACTIVE`, `suspendedUntil = null`

---

## 5. 신고 처리 시 자동 제재 (applySanctionFromReport)

| ReportActionType | 동작 |
|------------------|------|
| WARN_USER | addWarning |
| SUSPEND_USER | addBan (현재 정지는 영구 차단으로 처리) |
| NONE, DELETE_CONTENT, OTHER | 제재 없음 |

---

## 6. 동시성 제어

- **경고 횟수**: `@Modifying @Query("UPDATE Users u SET u.warningCount = u.warningCount + 1 WHERE u.idx = :userId")` — DB 레벨 원자적 증가
