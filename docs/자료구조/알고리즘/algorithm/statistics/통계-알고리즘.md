# 통계 & 대시보드 알고리즘

## 1. 일별 통계 집계 (Daily Summary Pattern)

**목적**: 실시간 `COUNT(*)` 부하를 줄이기 위해 전날 데이터를 요약 저장

**위치**: `StatisticsScheduler.aggregateDailyStatistics()`

**스케줄**: `@Scheduled(cron = "0 30 18 * * ?")` — 매일 18:30

---

## 2. 집계 항목 (aggregateStatisticsForDate)

| 항목 | 쿼리 |
|------|------|
| newUsers | `usersRepository.countByCreatedAtBetween` |
| newPosts | `boardRepository.countByCreatedAtBetween` |
| newCareRequests | `careRequestRepository.countByCreatedAtBetween` |
| completedCares | `careRequestRepository.countByDateBetweenAndStatus(..., COMPLETED)` |
| totalRevenue | 0 (고정) |
| activeUsers | `usersRepository.countByLastLoginAtBetween` |
| newMeetups | `meetupRepository.countByCreatedAtBetween` |
| meetupParticipants | `meetupParticipantsRepository.countByJoinedAtBetween` |
| newReports | `reportRepository.countByCreatedAtBetween` |

---

## 3. 중복 방지

- `dailyStatisticsRepository.findByStatDate(date).isPresent()` → 이미 있으면 건너뜀

---

## 4. Backfill (과거 데이터 초기화)

- `backfillStatistics(days)` — 오늘 기준 `days`일 전부터 어제까지 순차 집계
