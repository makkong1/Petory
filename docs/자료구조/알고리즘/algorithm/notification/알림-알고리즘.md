# 알림 시스템 알고리즘

## 1. 이중 저장소 전략

- **Redis**: 최신 알림 50개, TTL 24시간 — 실시간 조회 성능
- **MySQL**: 전체 알림 영구 저장 — 이력 관리

---

## 2. 알림 생성 시

1. `Notification` 저장 (MySQL)
2. `saveToRedis(userId, dto)` — Redis에 추가
3. `sseService.sendNotification(userId, dto)` — SSE 실시간 전송

---

## 3. Redis 저장 규칙 (saveToRedis)

- 키: `notification:{userId}`
- 기존 리스트 앞에 새 알림 추가
- **최대 50개** 유지 → `subList(0, 50)`
- TTL: 24시간

---

## 4. 조회 시 병합 (mergeNotifications)

**위치**: `NotificationService.getUserNotifications()`

**흐름**:
1. Redis에서 조회
2. Redis에 데이터 있으면 → DB 조회
3. **병합**:
   - Redis ID Set 생성
   - DB 알림 중 Redis에 없는 것만 추가
   - `createdAt` 기준 최신순 정렬
4. Redis 없으면 → DB만 반환

---

## 5. 읽음 처리 시

- MySQL: `isRead = true`
- Redis: `removeFromRedis(userId, notificationId)` — 해당 알림 제거

---

## 6. 전체 읽음 처리 시

- MySQL: 해당 유저의 모든 미읽음 알림 `isRead = true`
- Redis: `notification:{userId}` 키 삭제
