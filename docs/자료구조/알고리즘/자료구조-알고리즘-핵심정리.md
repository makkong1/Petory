# 자료구조·알고리즘 핵심 정리 (CS)

면접·이론 복기용으로 **자주 나오는 구조와 알고리즘**만 압축했습니다. 증명은 생략하고, 복잡도·용도 중심입니다.

---

## Part 1. 자료구조 (Data Structures)

### 1.1 선형 구조

| 구조 | 특징 | 평균 시간 (참고) | 쓰는 경우 |
|------|------|------------------|-----------|
| **배열 Array** | 연속 메모리, 인덱스 접근 | 읽기/쓰기 O(1), 삽입·삭제 O(n) | 랜덤 접근, 캐시 친화 |
| **연결 리스트 Linked List** | 노드 + 포인터, 길이 가변 | 탐색 O(n), 앞쪽 삽입/삭제 O(1)* | 삽입/삭제 빈번, 크기 불명 |
| **스택 Stack** | LIFO | push/pop O(1) | DFS, 괄호 매칭, 실행 스택 |
| **큐 Queue** | FIFO | enqueue/dequeue O(1)* | BFS, 스케줄링, 버퍼 |
| **덱 Deque** | 양끝 삽입·삭제 | 양끝 O(1) | 슬라이딩 윈도우 최적화 |

\* 연결 리스트 구현 기준. 배열 기반 큐는 원형 버퍼로 amortized O(1).

### 1.2 해시 Hash Table

- **평균**: 탐색·삽입·삭제 O(1)  
- **최악**: 해시 충돌 시 O(n) (체이닝·개방주소법으로緩和)  
- **용도**: 빠른 조회, 카운팅, 중복 제거, 캐시 키  
- **주의**: 순서 보장 X(LinkedHashMap 등 별도), 좋은 해시 함수·로드 팩터

### 1.3 트리 Tree

| 구조 | 특징 | 연산 (균형 시) |
|------|------|----------------|
| **이진 탐색 트리 BST** | 왼쪽 < 루트 < 오른쪽 | 탐색·삽입 평균 O(log n), 편향 시 O(n) |
| **균형 트리** (AVL, Red-Black 등) | 높이 O(log n) 보장 | 탐색·삽입·삭제 O(log n) |
| **힙 Heap** | 완전이진트리 + 힙 속성 | 최소/최대 추출 O(log n), peek O(1) |
| **트라이 Trie** | 문자열 접두사 공유 | 길이 L이면 검색 O(L) |

- **BST 활용**: 정렬된 순서 순회, 범위 검색  
- **힙 활용**: 우선순위 큐, Top-K, 스케줄링  
- **세그먼트 트리 / 펜윅** (심화): 구간 합·최소 등 쿼리 O(log n)

### 1.4 그래프 Graph

- **표현**: 인접 리스트(희소 그래프에 유리), 인접 행렬(밀집·간선 존재 여부 O(1))  
- **방향/무방**, **가중치** 여부에 따라 알고리즘 선택

### 1.5 기타

| 구조 | 용도 |
|------|------|
| **Union-Find (Disjoint Set)** | 연결 요소, Kruskal MST — `find`/`union` 거의 O(α(n)) |
| **Bloom Filter** | 멤버십 대략적 판정, 메모리 절약 (오탐 가능) |

## Part 2. 알고리즘 (Algorithms)

### 2.1 정렬 Sorting

| 알고리즘 | 최선 | 평균 | 최악 | 안정 | 비고 |
|----------|------|------|------|------|------|
| 버블/선택/삽입 | O(n) | O(n²) | O(n²) | 삽입만 안정 가능 | 교육용·작은 n |
| 머지 소트 | O(n log n) | O(n log n) | O(n log n) | 안정 | 추가 메모리 O(n) |
| 퀵 소트 | O(n log n) | O(n log n) | O(n²) | 불안정 | 실무에서 많이 씀 |
| 힙 소트 | O(n log n) | O(n log n) | O(n log n) | 불안정 | in-place |
| 카운팅/기수 | O(n+k) | — | — | — | 정수·범위 작을 때 |

- **Java `Arrays.sort`**: 객체는 TimSort(안정), 기본형은 듀얼 피벗 퀵소트 등.

### 2.2 이진 탐색 Binary Search

- **조건**: 정렬된 배열/구간  
- **시간**: O(log n)  
- **변형**: 하한/상한(lower_bound), 파라메트릭 서치(최소 x where 조건 만족)

### 2.3 그래프 탐색

| 알고리즘 | 자료구조 | 시간 | 용도 |
|----------|----------|------|------|
| **DFS** | 스택(재귀) | O(V+E) | 연결성, 사이클, 위상정렬 보조 |
| **BFS** | 큐 | O(V+E) | 최단거리(가중치 1), 레벨 순회 |

### 2.4 최단 경로·MST (가중 그래프)

| 알고리즘 | 조건 | 시간(대략) |
|----------|------|------------|
| **다익스트라 Dijkstra** | 비음수 가중치 | O((V+E) log V) — 힙 사용 시 |
| **벨만-포드 Bellman-Ford** | 음수 가중치 허용 | O(VE) |
| **플로이드-워셜** | 전쌍 최단거리 | O(V³) |
| **크루스칼 / 프림** | MST | 간선 정렬·Union-Find 등 |

### 2.5 패턴

| 패턴 | 아이디어 | 예시 |
|------|----------|------|
| **투 포인터 Two Pointers** | 정렬 배열에서 두 인덱스 이동 | 합 문제, 팰린드롬 |
| **슬라이딩 윈도우** | 고정/가변 구간을 O(n)으로 | 연속 부분합, 최대/최소 |
| **그리디 Greedy** | 매 단계 최선 선택 → 전역 최적은 문제 의존 | 활동 선택, 동전(특수) |
| **동적계획법 DP** | 중복 부분문제 + 최적 부분구조 | 피보나치, 배낭, LCS |

### 2.6 재귀·분할정복

- **마스터 정리**: T(n) = aT(n/b) + O(n^c) 형태의 점근 복잡도 분석  
- **분할정복**: 머지소트, 이진탐색, 일부 DP

## Part 3. 복잡도 한눈에

- **O(1)**: 해시 맵 평균 조회, 스택 push  
- **O(log n)**: 이진탐색, 균형 BST, 힙 연산  
- **O(n)**: 배열 한 번 순회, 해시 전체  
- **O(n log n)**: 효율적 정렬, 많은 “정렬 후 처리”  
- **O(n²)**: 이중 루프, 단순 비교  
- **O(2^n)**: 부분집합 전수, 무지성 재귀 (가지치기 필요)

**공간**: 입력 제외 추가 메모리 — 보조 배열 O(n), 재귀 깊이 O(n) 등.

## Part 4. Petory 백엔드 — 자료구조 ↔ Java 컬렉션 (실사용)

아래는 `backend/main/java` 기준으로 **어떤 인터페이스·구현체를 자주 쓰는지**를 정리한 것입니다. (코드 검색 기준, 시점에 따라 증감 가능)

### 4.1 요약: “많이 쓴 순” 느낌

| 순위 | 타입 | CS 대응 | 비고 |
|------|------|---------|------|
| 1 | **`List`** (인터페이스) + 구현은 대부분 **`ArrayList`** 또는 Stream `collect` | 동적 배열 | DTO·리포지토리 반환·엔티티 `@OneToMany`에 전부 등장 |
| 2 | **`Map`** + 구현 **`HashMap`** | 해시 테이블 | API 응답 조립, 그룹핑, 에러 바디, 네이버/지오코딩 응답 가공 |
| 3 | **`Optional`** | null 안전 (컬렉션은 아님) | Spring Data `findById` 등 단건 조회에 사실상 표준 |
| 4 | **`Stream` + `Collectors`** | (내부적으로 List/Map 할당) | `toList()`, `groupingBy`, `toMap` — 서비스·컨버터 전역 |
| 5 | **`ConcurrentHashMap`** | 스레드 안전 해시 | SSE 연결·OAuth 등 **멀티스레드 공유 맵**이 필요한 소수 코드 |
| 6 | **`LinkedHashMap`** | 삽입 순서 보장 해시 | 거의 없음(예: OAuth·위치 추천 등 **순서 유지**가 필요할 때만) |
| 7 | **`HashSet` / `Set`** | 집합·중복 제거 | `HashSet`은 공공데이터 **중복 키 제거** 등, `Set.of`는 **허용 MIME/확장자** 같은 작은 불변 집합 |

**현재 백엔드에서 검색 기준 거의 안 쓰는 것**: `TreeMap` / `TreeSet`(정렬 맵·셋), `PriorityQueue`, `ArrayDeque`, `LinkedList` — 우선순위 큐·덱·연결리스트 기반 알고리즘은 이 코드베이스에서 직접 쓰지 않음.

### 4.2 용도별로 보면

| 목적 | 흔한 선택 | Petory 예시 |
|------|-----------|-------------|
| 순서 있는 나열·페이징 결과 | `List` / `ArrayList` | 게시글·댓글·케어 요청 DTO, `ActivityService`에서 결과 누적 |
| ID → 객체 / 키 → 값 | `HashMap` | `BoardService`, `CommentService` 등에서 ID 맵으로 묶어 한 번에 변환 |
| HTTP/JSON 응답 맵 | `HashMap<String, Object>` | `AuthController`, `GlobalExceptionHandler`, `MeetupController`, 지오코딩 응답 |
| JPA 일대다 | `List<엔티티>` | Hibernate가 `PersistentBag` 등으로 매핑 — **자바 관점에서는 List** |
| 동시 접근 맵 | `ConcurrentHashMap` | `OAuth2DataCollector`, `NotificationSseService` |
| 중복 제거 | `HashSet` | `PublicDataLocationService` 배치 처리 시 키 중복 방지 |
| 소량 불변 집합 | `Set.of(...)` | `FileStorageService` 허용 Content-Type / 확장자 |
| 빈/단일 불변 리스트 | `Collections.emptyList()`, `Collections.singletonList()` | 방어적 반환·어댑터 |

### 4.3 이론(Part 1~2)과의 대응

- **배열·동적 배열**: 실무에서는 `ArrayList` + `List` 인터페이스가 기본.  
- **해시**: 조회·그룹핑·캐시형 맵은 `HashMap`. 순서가 필요하면 `LinkedHashMap`.  
- **트리 기반 정렬 맵/우선순위 큐**: 이 프로젝트 백엔드에서는 **직접 사용 빈도 낮음** — 정렬은 주로 **DB `ORDER BY`**·**스트림 `sorted()`** 로 처리하는 편.
