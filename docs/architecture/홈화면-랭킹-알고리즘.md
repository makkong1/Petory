# 홈 화면 랭킹 알고리즘

**작성일**: 2026-05-14  
**구현 브랜치**: feat-homepage-ranking-algorithm → dev

---

## 개요

홈 화면 4개 탭(주변서비스 / 모임 / 실종신고 / 커뮤니티)에서 상위 6개 항목을 어떤 기준으로 선정하는지 정의한다.

---

## 1. 주변서비스 탭

### 엔드포인트
`GET /api/location-services/search?sort=score&size=6`

### 알고리즘 — 복합 스코어

```
score = 0.5 × rating × log10(reviewCount + 1)
      + 0.2 × petFriendly (1.0 or 0.0)
```

| 항목 | 가중치 | 설명 |
|---|---|---|
| `rating × log10(reviewCount+1)` | 0.5 | 평점 × 리뷰 신뢰도. log로 리뷰 수 폭발 방지 |
| `petFriendly` | 0.2 | 반려동물 동반 가능 장소 우대 |

### 스코어 갱신 주기
- **배치**: 매일 자정 `LocationServiceScoreScheduler`가 전체 2만건 재계산
- **컬럼**: `locationservice.score DOUBLE DEFAULT 0.0`

### 폴백
- `sort=score`가 없으면 `rating DESC`

### 관련 파일
- `domain/location/entity/LocationService.java` — score 필드
- `domain/location/service/LocationServiceService.java` — post-sort 로직
- `domain/location/service/LocationServiceScoreScheduler.java` — 배치

## 2. 모임 탭

### 엔드포인트
`GET /api/meetups/home?lat={lat}&lng={lng}&size=6`

### 알고리즘 — 복합 스코어

```
score = 0.4 × distScore
      + 0.4 × urgencyScore
      + 0.2 × capacityScore
```

| 항목 | 가중치 | 공식 | 설명 |
|---|---|---|---|
| distScore | 0.4 | `max(0, 1 - distKm / 50)` | 반경 50km 이내, 가까울수록 1에 근접 |
| urgencyScore | 0.4 | `max(0, 1 - daysUntilMeetup / 30)` | 30일 이내 모임, 임박할수록 1에 근접 |
| capacityScore | 0.2 | `1 - currentParticipants / maxParticipants` | 빈 자리가 많을수록 1에 근접 |

### 처리 흐름

```
lat/lng 있음
  → getNearbyMeetups(lat, lng, 50km, size×3) 후보 조회
  → status=RECRUITING 필터
  → 복합 스코어 계산 및 정렬
  → 상위 6개 반환

lat/lng 없음 또는 후보 0개
  → getAvailableMeetups(date ASC, size=6) 폴백
```

### 관련 파일
- `domain/meetup/controller/MeetupController.java` — `/home` 엔드포인트
- `domain/meetup/service/MeetupService.java` — `getHomeMeetups()`

## 3. 실종신고 탭

### 엔드포인트
`GET /api/missing-pets/home?lat={lat}&lng={lng}&size=6`

### 알고리즘 — 복합 스코어

```
score = 0.6 × recencyScore
      + 0.4 × distScore
```

| 항목 | 가중치 | 공식 | 설명 |
|---|---|---|---|
| recencyScore | 0.6 | `max(0, 1 - daysSinceLost / 14)` | 실종 후 2주 이내, 최근일수록 1에 근접 |
| distScore | 0.4 | `max(0, 1 - distKm / 20)` | 반경 20km 이내, 가까울수록 1에 근접 |

### 처리 흐름

```
lat/lng 있음
  → status=MISSING 최신 50개 조회
  → 복합 스코어 계산 및 정렬
  → 상위 6개 반환

lat/lng 없음
  → lostDate DESC LIMIT 6 폴백
```

### 관련 파일
- `domain/board/controller/MissingPetBoardController.java` — `/home` 엔드포인트
- `domain/board/service/MissingPetBoardService.java` — `getHomeMissing()`

## 4. 커뮤니티 탭

### 엔드포인트
`GET /api/boards/popular?period=WEEKLY`

### 알고리즘 — 인기 점수 (기존 구현)

```
score = (likeCount × 3) + (commentCount × 2) + viewCount
```

### 스냅샷 조회 우선순위
1. 정확한 날짜 매칭 스냅샷
2. 기간 겹침 스냅샷
3. 최근 스냅샷 (Top 30)
4. 스냅샷 없으면 → 최신 게시글 10개로 폴백 (Step 1에서 추가)

### 배치
매일 자정 `BoardPopularityScheduler`가 WEEKLY/MONTHLY 스냅샷 생성

### 관련 파일
- `domain/board/service/BoardPopularityService.java`

## 위치 좌표 흐름 (프론트엔드)

```
HomePage 마운트
  → navigator.geolocation.getCurrentPosition()
  → 성공: userCoords = { lat, lng }
  → 실패/미지원: userCoords = null

탭 전환 시 fetchTabData()
  → userCoords 있으면 lat/lng 전달
  → 없으면 null 전달 (서버에서 폴백 처리)
```

### 관련 파일
- `frontend/src/components/Home/HomePage.js`
- `frontend/src/api/meetupApi.js` — `getHomeMeetups(lat, lng, size)`
- `frontend/src/api/missingPetApi.js` — `getHomeMissing(lat, lng, size)`
