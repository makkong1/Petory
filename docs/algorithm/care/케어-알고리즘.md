# 펫 케어 & 매칭 알고리즘

## 1. 상태 전이 (CareRequestStatus)

**상태**: `OPEN` → `IN_PROGRESS` → `COMPLETED` / `CANCELLED`

| 상태 | 의미 |
|------|------|
| OPEN | 모집 중, 지원자 대기 |
| IN_PROGRESS | 매칭 완료, 케어 진행 중 |
| COMPLETED | 케어 완료 |
| CANCELLED | 취소 |

**전이 조건** (CareRequestService.updateStatus):
- **작성자** 또는 **승인된 제공자**만 상태 변경 가능 (관리자 제외)
- `COMPLETED` 시 → 에스크로에서 제공자에게 코인 지급
- `CANCELLED` 시 → 에스크로에서 요청자에게 환불

---

## 2. 에스크로 연동 흐름

**거래 확정** (채팅에서 확정):
- `ConversationService` → `PetCoinEscrowService.createEscrow()`
- 요청자 코인 차감 → 에스크로 HOLD 생성

**케어 완료**:
- `CareRequestService.updateStatus(..., COMPLETED)`
- `PetCoinEscrowService.releaseToProvider()` → HOLD → RELEASED

**케어 취소**:
- `CareRequestService.updateStatus(..., CANCELLED)`
- `PetCoinEscrowService.refundToRequester()` → HOLD → REFUNDED

**동시성**: `findByCareRequestForUpdate` (비관적 락)으로 중복 지급/환불 방지

---

## 3. 생애주기 관리

```
요청 생성 → 지원 → 매칭(거래 확정) → 케어 수행 → 리뷰
```

- **채팅 연동**: 케어 요청 생성 시 1:1 채팅방 자동 생성
- **매칭**: 채팅에서 거래 확정 시 CareApplication ACCEPTED, CareRequest IN_PROGRESS
