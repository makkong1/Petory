# 산책 & 오프라인 모임 알고리즘

## 1. Haversine 반경 검색

**목적**: 사용자 위치 기준 반경(km) 내 모임 조회

**위치**: `SpringDataJpaMeetupRepository.findNearbyMeetups()`

**쿼리**: Native SQL
- `lat ± radius/111.0` 범위로 위도 필터
- `lng ± radius/(111*cos(radians(lat)))` 범위로 경도 필터
- Haversine: `6371 * acos(cos(radians(lat))*cos(radians(m.latitude))*cos(radians(m.longitude)-radians(lng)) + sin(radians(lat))*sin(radians(m.latitude))) <= radius`
- `date > currentDate` — 미래 모임만
- `status != 'COMPLETED'` — 종료된 모임 제외

**정렬**: 거리 오름차순, 날짜 오름차순

## 2. 인원 관리 (원자적 증가)

**목적**: 최대 인원 초과 방지

**위치**: `MeetupRepository.incrementParticipantsIfAvailable()`

**쿼리**:
```sql
UPDATE Meetup m SET m.currentParticipants = m.currentParticipants + 1
WHERE m.idx = :meetupIdx AND m.currentParticipants < m.maxParticipants
```

**반환**: 업데이트된 행 수 (0 또는 1)
- **0**: 인원이 가득 참 → `MeetupConflictException.fullCapacity()`
- **1**: 성공 → 참가자 추가 후 `entityManager.refresh(meetup)`

**참가 시**: 주최자는 이미 참가자로 포함되어 있으므로 `incrementParticipantsIfAvailable` 호출 안 함
