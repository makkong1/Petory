# B — 트렌드 카테고리 페이지 + 시계열

- **일자**: 2026-05-13
- **상위 컨텍스트**: A(카드 강화) 직후 진행. 가이드 §M2 옵션("관리자 대시보드에 시계열 차트 추가") 을 사용자용으로 확장해서, **비-시설 카테고리(`snack`/`food`/`clothes`)의 트렌드를 일반 사용자에게 처음 노출** 한다.
- **가이드**: [`../PETORY-INTEGRATION.md`](../PETORY-INTEGRATION.md) §5(트렌드 시계열), §M2

## 1. 왜 (배경)

가이드 §7.1 표:

| context | 의미 | 시설 타입 | 트렌드 카테고리 |
|---|---|---|---|
| `grooming` | 미용 | BUSINESS | grooming |
| `hospital` | 병원 | HOSPITAL | hospital |
| **`snack`** | 간식 | (없음) | snack |
| **`food`** | 사료 | (없음) | food |
| **`clothes`** | 의류 | (없음) | clothes |

`snack`/`food`/`clothes` 는 **공인 시설이 없는 카테고리** — 가이드 표현 그대로 "트렌드 중심" 으로 구성하라고 명시되어 있습니다. 그런데 Petory 에는 추천 카드 외 트렌드만 보여주는 화면이 없어, 사용자가 "요즘 어떤 사료가 인기인지" 같은 정보에 도달할 경로가 막혀 있었습니다.

B 의 목표는 **트렌드 자체를 콘텐츠로 만든다** — 시설이 없어도 가치 있는 화면 한 장.

### 3.3 프론트 — pivot 헬퍼

서버는 `(date ASC, score DESC)` 정렬로 `[{date, keyword, score}]` 형식을 주는데, recharts 의 `<LineChart>` 는 `data={[{date, kwA: 32, kwB: 28}, ...]}` 형식을 받아 키워드별 `<Line dataKey="kwA">` 를 그립니다. 변환 한 번 필요.

```32:62:frontend/src/components/Recommendation/TrendCategoryPage.js
function pivotPoints(points) {
  // points: [{date, keyword, score}, ...] (date ASC, score DESC by 서버 정렬)
  // → recharts 가 쓰기 좋은 형태로 피벗:
  //   chartRows: [{ date, keywordA: 32, keywordB: 28, ... }, ...]
  //   keywords:  ["keywordA", "keywordB", ...] (시리즈 정의용, 등장 빈도순)
  if (!points || points.length === 0) {
    return { chartRows: [], keywords: [] };
  }

  const rowsByDate = new Map();
  const keywordFrequency = new Map();

  for (const p of points) {
    if (!rowsByDate.has(p.date)) {
      rowsByDate.set(p.date, { date: p.date });
    }
    rowsByDate.get(p.date)[p.keyword] = p.score;
    keywordFrequency.set(p.keyword, (keywordFrequency.get(p.keyword) || 0) + 1);
  }

  const chartRows = Array.from(rowsByDate.values()).sort((a, b) =>
    a.date.localeCompare(b.date)
  );
  // 등장 빈도 많은 키워드부터 (legend 가 보기 좋아짐)
  const keywords = Array.from(keywordFrequency.entries())
    .sort((a, b) => b[1] - a[1])
    .map((e) => e[0]);

  return { chartRows, keywords };
}
```

## 6. 의도적으로 안 한 것

| 항목 | 이유 |
|---|---|
| 단순 `GET /trends/{category}` 프록시 | 현재 시점 키워드는 추천 카드의 `data.trends` 로 이미 노출 중. 중복. |
| Navigation 메뉴에 "트렌드" 추가 | 사용자 동선상 추천 카드에서 진입이 자연스러움. 메뉴 추가는 IA(정보구조) 변경이라 외과적 범위 밖. |
| 차트에 5개 초과 키워드 표시 | 라인이 5개 넘으면 가독성 ↓. "상위 5개" 안내 텍스트로 사용자에게 명시. |
| 카테고리 직접 URL 진입 (`/trends/snack` 같은 라우팅) | 앱이 `activeTab` state 기반이라 URL 라우팅이 별도 작업 필요. 현재는 무관한 변경이 너무 많이 들어가 보류. |
| 관리자 대시보드용 별도 시계열 차트 | 사용자용 페이지가 이미 시계열을 보여주므로, 같은 화면을 관리자도 사용 가능. 별도 화면은 needs 가 나오면 추가. |
